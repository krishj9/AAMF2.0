The platform uses a serverless architecture on AWS with the following components:

  • Angular Frontend hosted on S3 static website
  • API Gateway HTTP API for routing requests
  • Lambda Functions running FastAPI services
  • DynamoDB for durable workflow and portfolio persistence
  • Bedrock for LLM-powered agent capabilities
  • AgentCore for memory and observability

Environment Model

Each environment has isolated configuration, IAM, data boundaries, and deployment artifacts:

dev test staging main

This ensures safe promotion paths and prevents cross-environment data leakage.

Configuration Baseline

Key Configuration Parameters

  • Schema Version: Ensures data contract compatibility
  • Policy Version: Tracks risk compliance rules
  • Model Configuration: Bedrock model selection and guardrails
  • Tracing Provider: bedrock_agentcore or langsmith
  • Timeout/Retry Budgets: Lambda execution limits
  • Market Stream Limit: MARKET_STREAM_MAX_EVENTS for bounded streams

Infrastructure as Code

All AWS resources are managed with Terraform:

  • Reusable modules under infra/terraform/modules/
  • Environment-specific inputs under infra/terraform/environments/<env>/
  • Example variable files as *.tfvars.example
  • Always run terraform plan before terraform apply

⚠️ Important

Never commit:

  • .tfstate files (contain sensitive data)
  • .terraform/ directories (local cache)
  • Real *.tfvars files (contain secrets)

DynamoDB Runtime Selection

Local Development

Uses DynamoDB Local at http://localhost:55000

Perfect for testing without AWS costs

AWS Deployment

Uses hosted DynamoDB by leaving endpoint unset

Automatic scaling and high availability

Design Principle: The application repository layer must not branch business logic between local and AWS modes. Configuration is environment-based only.

Lambda Packaging

Build packages before Terraform plan/apply:

./infra/scripts/package_lambda.sh

Packaging Details

  • Packages written to infra/build/ (not committed)
  • Each FastAPI service has a Mangum handler for API Gateway events
  • Uses AWS SAM Python 3.13 Docker image for consistent native wheels
  • For large dependencies, switch to container-image Lambdas using ECR

Frontend Hosting

The Angular frontend is hosted on S3 as a static website:

  • S3 bucket provisioned per environment
  • Runtime configuration via app-config.js
  • API Gateway URL injected before upload
  • CloudFront optional for HTTPS and custom domains

Deployment Script

./infra/scripts/publish_frontend.sh <api_url> <bucket_name>
